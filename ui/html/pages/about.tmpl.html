About

Hi, I'm Lyndon.

I’m a software engineer with a keen interest in backend technologies, and a particular fondness for Go. I am analytical, love problem-solving, and enjoy thinking about systems and scale. My career has been non-linear, and includes roles such as strength coach, data analyst, and service coordinator for mental health services. This broad range of experience has been invaluable for my development.

Technical Interests

Go

When programming, I'm having the most fun (and doing my best work) when using a statically-typed, lower-level language. My introduction to programming was C, and that essence has always stayed with me. I'm drawn to Go as it allows me to handle the lower-level operations that I find most interesting about programming, whilst being remarkably productive, performant, and does not abuse abstraction. My goal is to become a wizard at Go.

Data Structures & Algorithms

I’m drawn to data structures and algorithms because they are the bedrock upon which efficient software is built. All other things being equal, faster software, that uses less resources, is better. Many modern technologies try to prevent programmers from needing to think about memory within the computer, or how costly it will be to add an element collection; but I think these are important. A developer can still make tremendous progress on what they are building while taking even just some care along the way to make sure their solution is efficient.

Me, The Person
